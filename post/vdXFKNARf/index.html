    • 全手动生成服务证书

    使用cfssl生成服务证书,用于kubernetes搭建和etcd集群搭建

    证书生成步骤

    主要步骤如下：

    • 安装证书工具
    • 创建签名请求文件
    • 生成相应的ca证书和对应服务证书

    下面以为harbor服务为例，生成harbor服务证书

    安装证书工具

    wget https://pkg.cfssl.org/R1.2/cfssl_linux-amd64
chmod +x cfssl_linux-amd64
mv cfssl_linux-amd64 /usr/local/bin/cfssl

wget https://pkg.cfssl.org/R1.2/cfssljson_linux-amd64
chmod +x cfssljson_linux-amd64
mv cfssljson_linux-amd64 /usr/local/bin/cfssljson

wget https://pkg.cfssl.org/R1.2/cfssl-certinfo_linux-amd64
chmod +x cfssl-certinfo_linux-amd64
mv cfssl-certinfo_linux-amd64 /usr/local/bin/cfssl-certinfo

    创建签名请求文件

    1. 生成ca配置文件
    mkdir /root/ssl && cd /root/ssl

cat > ca-config.json <<EOF
{
  "signing": {
    "default": {
      "expiry": "87600h"
    },
    "profiles": {
      "harbor": {
        "usages": [
            "signing",
            "key encipherment",
            "server auth",
            "client auth"
        ],
        "expiry": "87600h"
      }
    }
  }
}
EOF
    • 可以定义多个profiles，生成证书时候自动应用定义的profiles属性
    • signing 表示该证书可用于签名其它证书；生成的 ca.pem 证书中 CA=TRUE
    • server auth 表示client可以用该 CA 对server提供的证书进行验证
    • client auth 表示server可以用该CA对client提供的证书进行验证
    1. 生成ca签名请求文件
    cat > ca-csr.json <<EOF
{
  "CN": "harbor",
  "key": {
    "algo": "rsa",
    "size": 2048
  },
  "names": [
    {
      "C": "CN",
      "ST": "BeiJing",
      "L": "BeiJing",
      "O": "k8s",
      "OU": "System"
    }
  ],
    "ca": {
       "expiry": "87600h"
    }
}
EOF
    • 可根据自己的需求更改对应的值
    1. 生成服务签名请求文件

    创建服务签名文件 servicename-csr.json

    cat > harbor-csr.json <<EOF
{
    "CN": "harbor",
    "hosts": [
      "127.0.0.1",
      "100.64.16.179"
    ],
    "key": {
        "algo": "rsa",
        "size": 2048
    },
    "names": [
        {
            "C": "CN",
            "ST": "BeiJing",
            "L": "BeiJing",
            "O": "k8s",
            "OU": "System"
        }
    ]
}
EOF
    • 替换和增加hosts字段为自己服务的ip地址或者域名
    • 替换CN等字段为自己的服务名

    生成ca和harbor证书

    cfssl gencert -initca ca-csr.json | cfssljson -bare ca
cfssl gencert -ca=ca.pem -ca-key=ca-key.pem -config=ca-config.json -profile=harbor harbor-csr.json | cfssljson -bare harbor
    • 替换profile为自己在ca-config.json定义的profiles
    • 替换harbor-csr.json为自己创建的service-csr.json
    • 替换harbor为自己的服务名

    导入证书到客户端

    拷贝ca.pem证书到客户端节点，区分环境执行以下操作

    1. Red Hat (CentOS etc)
    cat ca.pem >> ca.crt 
cp ca.crt /etc/pki/ca-trust/source/anchors/ca.crt
update-ca-trust
    1. Ubuntu
    cat ca.pem >> ca.crt 
cp ca.crt  /usr/local/share/ca-certificates/ca.crt 
update-ca-certificates
